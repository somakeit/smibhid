SMIBHID - API

Endpoints

Endpoint Method Options Description
/api/wlan/mac GET Get WLAN MAC address
/api/version GET Get firmware version
/api/hostname GET Get smibhid hostname
/api/firmware_files GET, POST
  • List files: GET
  • Add file: POST: Submit Value = "Add", URL = Str: id = "url"
  • Remove file: POST: Submit Value = "Remove", Str: URL
List, add or remove URLs staged for download and patch on reset
/api/reset POST Reset SMIBHID
/api/sensors/modules GET List sensor modules
/api/sensors/modules/{module} GET Insert module name from module list in {module} parameter List module sensors
/api/sensors/readings/latest GET Get latest sensor readings from all modules
/api/sensors/readings/log/{log_type} GET Insert log type in {log_type} parameter: minute or hour Get last 60 minutes (minute) or hourly roll up log (hour) of sensor readings from all modules
/api/sensors/modules/scd30/auto_measure GET Get SCD30 measurement status. The SCD30 will not return sensible values until this returns 1 shortly after startup.
/api/sensors/modules/SCD30/auto_measure/{start_stop} PUT Insert start or stop in {start_stop} parameter to enable or disable automatic measurement. Put SCD30 measurement status. Without automatic measurement, the SCD30 will not take a measurement and get_readings will return 0.
/api/sensors/modules/SCD30/calibration/{ppm} PUT Insert integer ppm value to calibrate the SCD30 to in the {ppm} parameter. Enter 0 for default value specified in config file. Put SCD30 calibration value
/api/sensors/alarm/status GET Get the current status of the CO2 alarm. Returns a dictionary of status code and status text.
/api/sensors/alarm/statuses GET Returns a dictionary of possible alarm status codes mapped to their descriptions
/api/sensors/alarm/threshold GET Get the current CO2 alarm trigger threshold in PPM.
/api/sensors/alarm/reset_threshold GET Get the current CO2 alarm reset threshold in PPM.
/api/sensors/alarm/snooze_remaining GET Get the remaining snooze time for the CO2 alarm in seconds.
